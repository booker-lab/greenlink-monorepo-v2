# 🚚 Driver 앱 (배송기사)

## 🔔 서버 푸시 알림 (Level 2)
- **현재**: 로컬 알림만 동작 (앱 열려있을 때)
- **필요**: VAPID 키 쌍 생성 + 백엔드 `web-push` 라이브러리 연동
- **조건**: 백엔드 API 서버 구축 후 진행
- **비고**: `sw.js`의 push 이벤트 핸들러는 이미 준비됨

# 🔗 공통 (백엔드 / 인프라)

## 🖧 백엔드 API 서버 + DB 구축
- **현재**: 각 앱이 독립 메모리 상태로 실행 (Web↔Driver 데이터 공유 안 됨)
- **필요**: API 서버 (Node.js/Express 또는 Next.js API Routes) + DB (PostgreSQL/Supabase 등)
- **목표**: 소비자 주문 → 서버 저장 → Driver 앱 실시간 수신, 배송 상태 → Web 앱 반영
- **비고**: WebSocket 또는 SSE로 실시간 동기화 고려
